Here are some of the git commands that I find most useful:

Configuration

effect syntax example extra
change settings for all users git config --system [??].[attribute] [value] ... none yet ... affects file in /etc/gitconfig
change settings for one user git config --global [??].[attribute] [value] ... none yet ... affects file in $HOME/.gitconfig
change settings for one project git config [??].[attribute] [value] ... none yet ... affects file in [projectroot]/.git/config
change editor [none] git config core.editor emacs
check all settings git config --list ... none yet ...
check specific setting git config [??].[attribute] ... ? ...


Staging area

effect syntax example
remove file from staging area git rm --cached [file]
see diff of files in staging area git diff --staged
see tracked files git ls-files


History

effect syntax example extra
see a "pretty" commit history git log --pretty=[setting] git log --pretty=oneline setting: (oneline|short|full|fuller|format:"format string"). note that the --pretty option can interfere with other log options
see only a limited number of commits git log -[n] git log -3 accepts any (?) positive integer
show diffs with each commit git log -p git log -p
see history of a specific file git log --follow [file] can be combined with, say, -p for some really useful information!
see a branch graph git log --graph git log --graph
display file statistics with each commit: number of additions, deletions, etc. git log --stat see also --shortstat ??
more log options git log (--name-only|--name-status|--abbrev-commit|--relative-date|--since|--until|--grep|--no-merges)
make *all* criteria match git log --all-match [... conditions ...] instead of *any* predicate causing a match, *all* predicates must be true to match


Remote

effect syntax example extra
see remotes git remote -v
make a new remote git remote add [name] [location] [location] can (must?) be a url
get data from a remote -- everything that you don't yet have locally git fetch [remote]
show info about a remote git remote show [remote]
rename a remote (i.e. change the local alias -- doesn't change anything on the actual remote server) git remote rename [old] [new] also renames remote branch names
remove a remote git remote rm [remote]


Tags

effect syntax example extra
see all tags git tag -
see tags matching a pattern git tag -l [pattern] this doesn't seem to work
make a lightweight tag git tag [tagname] git tag v1.0.0
make an annotated tag git tag -a [tagname] -m 'commit message' what does '-a' do?
show a single tag git show [tagname] git show v1.0.0
make a signed tag git tag -s [tagname] -m 'commit message' uses GPG
verify a signed tag git tag -v [tagname] need signer's public key ... somewhere? in keyring?
make a tag of a previous commit git tag -a [tagname] [commit's sha-1 checksum]
push one tag to a remote git push [remote] [tagname] git push origin v1.0.0
push all tags to a remote git push [remote] --tags git push origin --tags


Miscellaneous

effect syntax example extra
amend previous commit. uses staging area for commit git commit --amend what happens with sha-1 business? does this command take more arguments (i.e. for commit message)?
make a command alias git config --global alias.[thealias] 'the original command' git config --global alias.mypush 'push origin master' ... not sure if those are supposed to be quotation marks or backticks or what ...
